# 협력사 전/출입 관리 시스템 Spec

## 1. 목적

- 사내 ITO 및 프로젝트에 투입되는 협력사 인력의 전출입 이력을 체계적으로 관리
- 신규 전입 인력에 대해 **보안서약서 서명**을 필수로 징구하고, 사내 시스템 접근을 위한 **AD 계정** 및 필요 시 **S-RBS** 계정 발급을 신청
- 인력 변동이력을 관리하여 그에 맞는 권한 변경/회수 조치를 적시에 수행

---

## 2. 기능 구성

좌측 메뉴 기준으로 아래와 같이 구성한다.

### 2-1) 마스터 관리

- 투입 과제명(ITO, 개별 과제)을 등록/수정/삭제하는 마스터 관리 기능
- 신청 화면의 과제명 드롭다운 등에서 참조

### 2-2) 신청

#### (1) 신규 신청 ★ 1차 구현 대상

**목적**

신규 투입 인력의 기본정보를 입력/저장하고, 보안서약서 발송 · AD 발급 · S-RBS 발급 신청을 수행하는 화면

**Grid 동작 요건**

| 항목 | 설명 |
|------|------|
| Excel 호환 복사/붙여넣기 | 기존 Excel Sheet에서 복사한 데이터를 Grid에 붙여넣기 시, 행 수만큼 자동으로 행이 생성되며 데이터가 채워짐 |
| Excel 유사 조작 | 키보드 방향키로 셀 이동, 셀 클릭으로 선택, 범위 선택(Shift+클릭/드래그) 후 복사/붙여넣기 지원 |
| 셀 편집 모드 | 셀 선택 후 Enter 입력 시 Edit 모드 진입 → 값 입력 후 Enter 시 Edit 모드 해제 |
| 초기 상태 | 최초 1행이 공란으로 존재 |

**Grid 상단 버튼**

| 버튼 | 동작 |
|------|------|
| 행 추가 | 하단에 빈 행 1개 추가 |
| 전체 선택 | 모든 행 선택 |
| 전체 삭제 | 모든 행 데이터 삭제 |

**행별 컨트롤**

- 각 행 맨 우측에 행 삭제 버튼 (해당 행만 삭제)

**신청 항목 (Checkbox, Grid 우측)**

| 항목 | 설명 |
|------|------|

**저장 동작**

1. 하단 "저장" 버튼 클릭
2. 확인 팝업: "신청 하시겠습니까?"
3. 확인 클릭 시 DB에 데이터 저장
4. 취소 클릭 시 팝업 닫힘, 동작 없음

## 3. 1차 구현 범위

> **2-2)-(1) 신규 신청** 화면의 Grid 기반 데이터 입력 → 수정 → 저장 기능을 우선 구현한다.

### 구현 항목

- [ ] 좌측 메뉴 네비게이션 (메뉴 구조만 잡고, 신규 신청만 활성)
- [ ] Excel 호환 Grid (복사/붙여넣기, 키보드 조작, 셀 편집)
- [ ] Grid 상단 버튼 (행 추가, 전체 선택, 전체 삭제)
- [ ] 행별 삭제 버튼
- [ ] 신청 항목 Checkbox (보안서약서, AD, S-RBS)
- [ ] 저장 버튼 + 확인 팝업 + DB 저장
- [ ] 백엔드 API (CRUD)
- [ ] DB 모델 (입력 + 시스템 컬럼)
